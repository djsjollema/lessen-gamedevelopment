### 1. Repository-beheerder (1 student)

Deze student: 1. Maakt een nieuwe GitHub-repository. 2. Voegt de
startbestanden toe (index.html, team.html, style.css). 3. Zet GitHub
Pages aan (Settings â†’ Pages). 4. Nodigt teamgenoten uit via **Settings â†’

------------------------------------------------------------------------

### 2. Issues maken & taken verdelen

Als groep: 1. Maak 5--10 issues voor onderdelen van de website, zoals: -
Header toevoegen - Teamsectie maken - Footer toevoegen - Styling
verbeteren - Contactpagina maken 2. Kies een issue en **assign** het aan
jezelf.

------------------------------------------------------------------------

### 3. Branch maken & aan je taak werken

Voor elke taak: 1. Maak een branch: `jouwnaam-feature`. 2. Pas de
bestanden aan volgens je issue. 3. Schrijf duidelijke commit-berichten.
4. Push je branch naar GitHub.

------------------------------------------------------------------------

### 4. Pull Request openen

1.  Open een Pull Request (PR) naar de `main` branch.
2.  Koppel de PR aan het issue.
3.  Vraag een teamgenoot om een review.

------------------------------------------------------------------------

### 5. Review & Merge

-   Bekijk elkaars PR's.
-   Geef feedback via comments.
-   Keur de PR goed en merge wanneer alles klopt.

------------------------------------------------------------------------

### 6. Conflicten oplossen (indien nodig)

Als twee personen hetzelfde bestand of dezelfde regel wijzigen, ontstaat
een **merge conflict**.\
Je lost dit op in de GitHub-editor door:

1.  "Incoming" en "Current" te vergelijken.
2.  De juiste regels te behouden.
3.  Het bestand op te slaan en op "Resolve" te klikken.

------------------------------------------------------------------------

### 7. Resultaat bekijken

Bekijk jullie website via GitHub Pages en bespreek: - Welke onderdelen
heeft ieder gemaakt? - Hoe verliep de samenwerking? - Wat heb je geleerd
van reviews en conflicten?
