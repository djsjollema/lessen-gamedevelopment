Team Oefening met GitHub

Werk samen in een team van 4 tot 6 studenten en leer hoe je GitHub gebruikt om samen te werken aan een project. Volg de stappen hieronder om als team een eindproduct te maken door middel van pull requests.

Stappenplan

  1. Maak een GitHub repository aan
    Eén student maakt een nieuwe repository aan en nodigt de andere teamleden uit als collaborators.
  2. Verdeel de taken
    Bepaal samen wie welke taak krijgt. Bijvoorbeeld:
    • Student 1: Ontwerp de header
    • Student 2: Schrijf de introductietekst
    • Student 3: Voeg een afbeelding of illustratie toe
    • Student 4: Maak een footer
    • Student 5: Voeg een extra sectie toe (optioneel)
    • Student 6: Controleer spelling en layout (optioneel)
  3. Maak een eigen branch
    Iedere student maakt een branch voor zijn/haar taak.
  4. Voer je taak uit
    Werk aan je onderdeel in je eigen branch en commit je wijzigingen.
  5. Maak een pull request
    Als je klaar bent, maak je een pull request naar main. Geef duidelijk aan wat je hebt toegevoegd.
  6. Review en merge
    Teamleden reviewen elkaars pull requests. Als alles klopt, wordt de pull request gemerged in main.
  7. Herhaal tot het eindproduct af is
    Herhaal deze stappen tot alle onderdelen zijn toegevoegd en het eindproduct klaar is.

Tips voor samenwerking

  • Communiceer duidelijk over wie wat doet.
  • Gebruik duidelijke commit messages.
  • Review elkaars werk en geef feedback.
  • Los merge conflicts samen op.

Samenwerken aan een Unity project

Werk als team van 4-6 studenten samen aan een Unity game of applicatie. Gebruik GitHub om code, assets en scenes te beheren en samen te voegen tot een eindproduct.

Stappenplan Unity samenwerking

  1. Maak een Unity project aan
    Eén student maakt een nieuw Unity project en zet dit in een GitHub repository. Nodig teamleden uit als collaborators.
  2. Verdeel de taken
    Bijvoorbeeld:
    • Student 1: Programmeert gameplay scripts
    • Student 2: Ontwerpt levels/scenes
    • Student 3: Maakt of verzamelt 2D/3D assets
    • Student 4: Regelt audio en muziek
    • Student 5: Test en documenteert (optioneel)
    • Student 6: UI/UX ontwerp (optioneel)
  3. Werk in branches
    Iedere student werkt in een eigen branch voor zijn/haar taak.
  4. Commit en push regelmatig
    Zorg dat je werk up-to-date blijft en push je wijzigingen naar GitHub.
  5. Maak pull requests
    Voeg je werk samen via pull requests naar main. Geef duidelijk aan wat je hebt toegevoegd of aangepast.
  6. Review en merge
    Teamleden reviewen elkaars werk en mergen als alles klopt.
  7. Let op Unity-bestanden
    Gebruik altijd .gitignore voor Unity en voorkom dat Library/ en Temp/ mappen in de repository komen. Werk bij voorkeur met YAML-format voor scenes en prefabs (instellen in Unity).

Tips voor Unity teams

  • Communiceer over wie aan welke scene of asset werkt.
  • Voorkom dat meerdere mensen tegelijk aan dezelfde scene werken.
  • Gebruik duidelijke commit messages en pull requests.
  • Los merge conflicts samen op, vooral bij scenes en prefabs.
  • Gebruik Unity .gitignore voor je repository.
