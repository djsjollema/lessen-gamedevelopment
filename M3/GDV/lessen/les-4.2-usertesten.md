# Les 4.2 — Usertesten

## Leerdoel

Na deze les kun je usertests uitvoeren en bruikbare feedback verzamelen.

---

## Theorie

### Waarom Usertesten?

| Jij denkt          | De speler ervaart    |
| ------------------ | -------------------- |
| "Dit is duidelijk" | "Wat moet ik doen?"  |
| "Dit is makkelijk" | "Dit is onmogelijk!" |
| "Dit is leuk"      | "Dit is frustrerend" |

**Je bent blind voor problemen in je eigen game.**

### Soorten Tests

| Type                | Wanneer      | Doel            |
| ------------------- | ------------ | --------------- |
| **Paper prototype** | Concept fase | Idee valideren  |
| **Playtest**        | Development  | Gameplay testen |
| **Usability test**  | Polish fase  | UX verbeteren   |

### Test Protocol

1. **Introduceer** — Leg kort uit wat je test (niet hoe het werkt!)
2. **Observeer** — Kijk en luister, help NIET
3. **Think aloud** — Vraag de tester hardop te denken
4. **Noteer** — Schrijf problemen op
5. **Debrief** — Stel vragen achteraf

### Goede Test Vragen

**Niet vragen:**

- "Vond je het leuk?" (sociaal wenselijk antwoord)
- "Was het duidelijk?" (ja/nee zegt weinig)

**Wel vragen:**

- "Wat probeerde je te doen toen X gebeurde?"
- "Wat verwachtte je dat er zou gebeuren?"
- "Wat was het meest frustrerende moment?"
- "Wat zou je als eerste veranderen?"

### Feedback Verwerken

| Observatie                | Probleem                       | Prioriteit | Oplossing           |
| ------------------------- | ------------------------------ | ---------- | ------------------- |
| Tester liep vast bij deur | Onduidelijk hoe deuren werken  | Hoog       | Tutorial toevoegen  |
| Tester miste power-up     | Power-up niet zichtbaar genoeg | Medium     | Groter maken + glow |

## Oefeningen

### Oefening 1: Test Voorbereiden

Bereid een playtest voor:

**1. Wat wil je testen?**
Kies 2-3 specifieke vragen:

- [ ] Zijn de controls duidelijk?
- [ ] Is de moeilijkheidsgraad goed?
- [ ] Begrijpen spelers het doel?
- [ ] Werkt de collision goed?
- [ ] ***

**2. Maak een observatie-template:**

| Tijd | Wat doet tester | Wat zegt tester | Probleem? |
| ---- | --------------- | --------------- | --------- |
| 0:00 |                 |                 |           |
| 0:30 |                 |                 |           |
| 1:00 |                 |                 |           |
| ...  |                 |                 |           |

**3. Bereid debrief-vragen voor:**

1. ***
2. ***
3. ***

---

### Oefening 2: Test Uitvoeren

Voer een 5-minuten playtest uit:

**Voorbereiding:**

- Zorg dat je game start zonder crashes
- Begin bij een logisch startpunt

**Tijdens de test:**

1. Laat een klasgenoot spelen (niet je duo-partner)
2. Zeg alleen: "Speel de game, denk hardop"
3. **Help niet!** Zelfs als ze vastlopen
4. Noteer alles wat opvalt

**Na de test:**

1. Stel je voorbereide vragen
2. Bedank de tester

**Verwerk de feedback:**

| Probleem | Hoe vaak gezien | Prioriteit (H/M/L) | Actie |
| -------- | --------------- | ------------------ | ----- |
|          |                 |                    |       |
|          |                 |                    |       |
|          |                 |                    |       |

## Toepassing

1. Voer deze week minimaal 2 playtests uit
2. Documenteer je bevindingen in je GDD of README
3. Kies de top 3 problemen om op te lossen
4. Bespreek met je duo-partner welke je eerst aanpakt

**Tip:** Test vroeg, test vaak. Een halve game testen is beter dan een "perfecte" game nooit testen.
