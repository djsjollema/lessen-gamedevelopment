## Theorie

### Game Analyse

Om een goede game te maken, moet je eerst begrijpen wat een game goed maakt. Analyseer bestaande games op:

| Aspect         | Vraag                                    |
| -------------- | ---------------------------------------- |
| **Core Loop**  | Wat doet de speler steeds opnieuw?       |
| **Doel**       | Wat probeert de speler te bereiken?      |
| **Uitdaging**  | Wat maakt het moeilijk?                  |
| **Feedback**   | Hoe weet de speler dat iets lukt/faalt?  |
| **Progressie** | Hoe wordt het spel interessant gehouden? |

### PAC-MAN Analyse

| Aspect     | PAC-MAN                           |
| ---------- | --------------------------------- |
| Core Loop  | Dots eten, ghosts ontwijken       |
| Doel       | Alle dots verzamelen              |
| Uitdaging  | Ghosts die je achtervolgen        |
| Feedback   | Score, geluidseffecten, power-ups |
| Progressie | Snellere ghosts, nieuwe levels    |

### Brainstormen

Bij het bedenken van je eigen variant:

- **Reskin:** Zelfde mechaniek, ander thema (bijv. onderwaterwereld)
- **Twist:** Voeg een nieuwe mechaniek toe (bijv. schieten)
- **Mashup:** Combineer met een andere game

## Toepassing

Bespreek je gameconcept met je duo-partner:

1. Pitch je idee in 30 seconden
2. Krijg feedback: wat is onduidelijk?
3. Schrijf een korte elevator pitch (2-3 zinnen)

**Voorbeeld elevator pitch:**

> "In mijn game speel je een robot die batterijen moet verzamelen in een fabriek. Andere robots proberen je te stoppen, maar met een EMP-powerup kun je ze tijdelijk uitschakelen."
